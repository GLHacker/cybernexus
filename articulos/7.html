Cómo configurar un laboratorio de pentesting con VirtualBox

Publicado el 26 Ene, 2025 por Admin

Practicar hacking en sistemas reales sin permiso es ilegal. La solución: crear tu propio laboratorio virtual seguro y legal. Aquí te enseño cómo.

Requisitos del Sistema

  • RAM: Mínimo 8GB (recomendado 16GB)
  • Procesador: CPU con virtualización (Intel VT-x o AMD-V)
  • Disco: 50GB libres mínimo
  • SO Host: Windows, macOS o Linux

Paso 1: Instalar VirtualBox

Descarga VirtualBox desde virtualbox.org. Es gratuito y open source.

# En Linux (Ubuntu/Debian)
sudo apt update
sudo apt install virtualbox virtualbox-ext-pack

Paso 2: Descargar Kali Linux

Ve a kali.org/get-kali y descarga la imagen para VirtualBox (archivo .ova).

Credenciales por defecto: Usuario: kali | Contraseña: kali

Paso 3: Importar Kali en VirtualBox

  1. Abre VirtualBox
  2. Archivo → Importar servicio virtualizado
  3. Selecciona el archivo .ova de Kali
  4. Asigna al menos 4GB de RAM y 2 CPUs
  5. Importar

Paso 4: Configurar la Red

Para aislar tu lab de internet:

  1. Selecciona la VM de Kali → Configuración → Red
  2. Adaptador 1: Red NAT (para acceso a internet)
  3. Adaptador 2: Red Interna (nombre: "pentest_lab")

Paso 5: Descargar Máquinas Vulnerables

Estas son máquinas intencionalmente vulnerables para practicar:

Principiantes

  • Metasploitable 2: sourceforge.net/projects/metasploitable
  • DVWA: Damn Vulnerable Web Application

Intermedios

  • VulnHub: vulnhub.com (cientos de VMs gratis)
  • HackTheBox: Exporta VMs de HTB

Paso 6: Importar Máquina Vulnerable

# Ejemplo con Metasploitable 2
1. Descargar el .zip
2. Extraer
3. VirtualBox → Nueva
4. Tipo: Linux, Versión: Ubuntu (64-bit)
5. Usar disco existente → Seleccionar .vmdk de Metasploitable
6. Red: Red Interna (pentest_lab)

Paso 7: Primera Prueba

Desde Kali, escanea la red interna:

sudo netdiscover -i eth1
# o
sudo nmap -sn 192.168.56.0/24

Deberías ver la IP de Metasploitable. ¡Ahora puedes empezar a hackearla legalmente!

Consejos de Seguridad

  • NUNCA conectes máquinas vulnerables directamente a internet
  • Usa snapshots antes de hacer cambios importantes
  • Mantén Kali actualizado: sudo apt update && sudo apt full-upgrade
